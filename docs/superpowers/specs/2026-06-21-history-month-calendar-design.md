# Календарь тренировок на странице «История»

## Цель

Добавить в верхнюю часть страницы «История» компактный календарь месяца, который показывает дни завершённых тренировок и позволяет переключать месяцы горизонтальным свайпом.

## Размещение и внешний вид

- Календарь является первым блоком содержимого страницы «История», перед блоками Backup, «Прогресс» и списком тренировок.
- Он не располагается в общей области сразу под навигационными вкладками.
- Визуальный язык повторяет календарь вкладки «Планы»: тёмная панель, проектная палитра, скругления, приглушённые соседние даты и выделение сегодняшнего дня.
- Календарь показывает шесть строк недель, чтобы высота блока не менялась между месяцами.
- Заголовок содержит название месяца и год.
- Дни недели идут с понедельника по воскресенье.
- Дни завершённых тренировок отмечаются небольшой точкой под числом.
- Даты соседних месяцев видимы, но приглушены.

## Поведение

- При открытии показывается текущий месяц.
- Горизонтальный свайп влево открывает следующий месяц, вправо — предыдущий.
- Жест считается переключением месяца только при явном горизонтальном движении; вертикальная прокрутка страницы должна сохраняться.
- Нажатие на день ничего не делает: календарь не фильтрует и не изменяет список истории.
- Переключение месяцев не ограничено историей тренировок.
- При смене календарного дня в открытом приложении выделение сегодняшней даты обновляется по существующему механизму календаря «Планы».

## Данные и границы

- Используется уже загруженный массив истории; новых API-маршрутов и запросов нет.
- Отметки строятся только для сессий со статусом `completed`.
- Дата тренировки определяется по локальной календарной дате `startedAt`, как в календаре «Планов».
- Несколько тренировок в один день дают одну точку.
- Некорректные даты игнорируются.

## Компоненты

- Добавить отдельный месячный календарь для истории, чтобы его навигационное состояние не смешивалось с недельным календарём «Планов».
- Общую логику форматирования локальной даты и обновления текущего дня переиспользовать из существующего календарного модуля без изменения его поведения.
- Подключить месячный календарь внутри `HistoryPanel` первым дочерним блоком.

## Доступность

- Секция календаря получает понятную русскую подпись с отображаемым месяцем.
- Каждая дата имеет доступное полное название; дата с тренировкой сообщает о наличии тренировки.
- Свайп дополняется компактными кнопками перехода к предыдущему и следующему месяцу для клавиатуры и устройств без удобного жеста.
- Кнопки имеют русские `aria-label` и достаточную область нажатия, не увеличивая визуальную высоту сетки.

## Проверка

- Модульные тесты: построение сетки 6 × 7, начало недели с понедельника, границы месяца и года, локальные даты, одна отметка для нескольких тренировок, игнорирование незавершённых сессий и некорректных дат.
- Компонентные тесты: заголовок месяца, доступные подписи, отметки тренировок и кнопки навигации.
- Тесты жеста: горизонтальный порог переключает месяц в нужную сторону, вертикальное движение не переключает.
- CSS-тесты: семь равных колонок, компактные точки, приглушённые соседние даты, стабильная шестинедельная сетка и сохранение вертикального скролла.
- Запустить typecheck и тесты web-приложения, затем общую сборку.
- Вручную проверить локальный мобильный viewport; горизонтальные жесты и прокрутку дополнительно проверить в Telegram на реальном телефоне.

## Вне задачи

- Фильтрация списка истории по выбранному дню или месяцу.
- Открытие тренировки по нажатию на дату.
- Изменения API, базы данных, Backup или прогресса.
