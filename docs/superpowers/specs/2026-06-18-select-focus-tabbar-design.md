# Сохранение нижнего меню при работе с выпадающими списками

## Проблема

Приложение скрывает нижнее меню, когда фокус находится на элементе, считающемся клавиатурным редактором. Сейчас к таким элементам ошибочно относится `select`. В Telegram WebView после закрытия системного списка `select` может сохранять фокус, поэтому нижнее меню остаётся скрытым до прокрутки или другого действия, вызывающего потерю фокуса.

## Решение

Исключить `select` из определения клавиатурного редактора. Нижнее меню продолжает скрываться при фокусе на:

- `input`;
- `textarea`;
- элементах с `contenteditable="true"`.

Фокус на любом `select`, включая выбор упражнения в блоке «Прогресс», не должен скрывать нижнее меню. Правило применяется ко всему приложению, поскольку нативные выпадающие списки не открывают экранную клавиатуру.

## Границы изменения

- Логика определения открытой клавиатуры по размеру viewport не меняется.
- Стили нижнего меню не меняются.
- Принудительный `blur` после выбора не добавляется.
- Backend и API не меняются.

## Проверка

Добавить чистую тестируемую функцию классификации активного элемента. Unit-тест должен подтвердить, что `input`, `textarea` и `contenteditable` скрывают нижнее меню, а `select` — нет. После реализации выполнить frontend-тесты и typecheck.

Реальное поведение закрытия нативного списка следует дополнительно проверить в Telegram iOS WebView, поскольку браузерные тесты не воспроизводят системный picker полностью.
