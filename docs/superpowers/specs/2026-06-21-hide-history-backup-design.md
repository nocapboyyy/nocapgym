# Скрытие Backup на странице «История»

## Цель

Убрать блок Backup из интерфейса страницы «История», сохранив возможность вернуть функцию позже без изменений backend API.

## Изменения UI

- Удалить JSX блока Backup из `HistoryPanel`.
- Удалить неиспользуемые `onExport` и `onImport` из props `HistoryPanel` и места его вызова.
- После месячного календаря сразу показывать блок «Прогресс», затем список истории.
- Не скрывать блок через CSS и не оставлять его в DOM.

## Что сохраняется

- Frontend-функции `exportBackup` и `importBackup` остаются в `App` без изменения поведения.
- Backend API экспорта и импорта, маршруты, схемы и данные остаются без изменений.
- Стили `.file-button` остаются, поскольку функция может вернуться в UI и их удаление не требуется для текущей задачи.

## Проверка

- Добавить server-render тест `HistoryPanel`, подтверждающий отсутствие Backup и сохранение календаря, прогресса и списка истории.
- Запустить web-тесты, web typecheck и полную сборку.
- Обновить `docs/ai/current-state.md`: экспорт/импорт поддерживается API/frontend-логикой, но временно не отображается в History UI.

## Вне задачи

- Удаление или изменение API Backup.
- Удаление frontend-функций импорта и экспорта.
- Перенос Backup на другую страницу или добавление feature flag.
