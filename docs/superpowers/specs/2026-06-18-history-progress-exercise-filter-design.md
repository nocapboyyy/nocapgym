# Фильтрация упражнений в блоке «Прогресс»

## Цель

Во вкладке «История» показывать в выпадающем списке блока «Прогресс» только упражнения, для которых существует хотя бы одна фактическая точка прогресса.

## Правило отбора

Упражнение включается в список, если в завершённой тренировке у него есть хотя бы один подход, который одновременно:

- имеет тип `working`;
- отмечен выполненным (`completed`);
- содержит фактический вес (`actualWeightKg` не `null`);
- содержит фактическое число повторений (`actualReps` не `null`).

Это правило соответствует текущему расчёту точек прогресса на backend.

## Реализация

Frontend вычисляет доступные упражнения из уже загруженного массива истории. Чистая функция проходит по упражнениям завершённых тренировок, проверяет наличие подходящего подхода, удаляет повторы по `exerciseId` и сортирует результат по названию.

Название и остальные данные упражнения берутся из вложенного объекта `exercise` в истории. Поэтому скрытое администратором упражнение остаётся доступным для просмотра старого прогресса. Если связанный объект упражнения отсутствует, элемент не добавляется: без названия он не может быть корректно представлен пользователю.

Новый API и изменения схемы данных не требуются. При отсутствии подходящих упражнений выпадающий список содержит только пункт «Выберите упражнение».

## Проверка

Unit-тест чистой функции должен подтвердить, что она:

- исключает упражнения без выполненных рабочих подходов с фактическими значениями;
- включает подходящие упражнения только один раз при наличии в нескольких тренировках;
- сохраняет скрытые упражнения из истории;
- возвращает упражнения в алфавитном порядке.

После реализации выполняются тесты и typecheck frontend.
